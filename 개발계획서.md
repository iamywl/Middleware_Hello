# 개발계획서

## Linux 기반 WEB/WAS 미들웨어 환경 구축 및 APM 모니터링 시스템

> 작성일: 2026-03-04
> 대상: 미들웨어(M/W) 기술엔지니어 직무 대비 실습 프로젝트

---

## 1. 프로젝트 개요

### 1.1 목적

채용공고 핵심 역량(WEB/WAS 운영, 미들웨어 구축, SSO/PKI 연동, APM 모니터링)을 직접 실습하여 실무 역량을 증명하는 포트폴리오를 구축한다.

### 1.2 시스템 아키텍처

```
[클라이언트 브라우저]
        |  HTTPS (SSL/TLS)
        v
[Nginx - 리버스 프록시 + 로드밸런싱]
       / \
      v   v
[Tomcat #1]  [Tomcat #2]  ← WAS 이중화
      \   /
       v v
[MySQL DB] + [Scouter APM Server] + [Keycloak SSO]
                    |
        [Grafana + Prometheus 대시보드]
```

### 1.3 기술 스택

| 분류 | 기술/도구 | 버전 |
|------|-----------|------|
| OS | Ubuntu | 22.04 LTS |
| WEB 서버 | Nginx | 1.24 |
| WAS | Apache Tomcat | 10.x (2대) |
| 언어/프레임워크 | Java 17, Spring Boot | 3.x |
| APM | Scouter | 최신 |
| 대시보드 | Grafana + Prometheus | 최신 |
| SSO | Keycloak | 최신 |
| 인증서 | OpenSSL (자체 CA) | - |
| 가상화 | Docker Compose | 최신 |
| DB | MySQL | 8.x |

## 3. 단계별 개발 계획

### Phase 1: 기본 인프라 구축 (1~2주차)

#### 목표
Nginx + Tomcat 이중화 환경에서 Spring Boot 앱이 정상 동작하는 상태 구현

#### 세부 작업

| 주차 | 작업 | 산출물 |
|------|------|--------|
| 1주차 | Docker Compose 기반 인프라 정의 (Ubuntu, Nginx, Tomcat x2, MySQL) | `docker-compose.yml` 초안 |
| 1주차 | Nginx 리버스 프록시 설정 (upstream 블록으로 Tomcat 연결) | `nginx.conf`, `default.conf` |
| 1주차 | Tomcat 2대 AJP/HTTP 커넥터 설정 | `server.xml` x2 |
| 2주차 | Nginx 로드밸런싱 구성 및 비교 테스트 (Round Robin, IP Hash) | 테스트 결과 문서 |
| 2주차 | Spring Boot WAR 패키징 및 Tomcat 배포 | 샘플 앱 WAR 파일 |
| 2주차 | 서버 시작/종료 스크립트 작성 | `scripts/` 디렉토리 |

#### 완료 기준
- 브라우저에서 접속 시 두 WAS에 요청이 번갈아 분산되는 것을 확인
- `curl` 테스트로 로드밸런싱 동작 검증 완료

---

### Phase 2: APM 모니터링 체계 구축 (3~4주차)

#### 목표
Scouter + Grafana로 WAS 및 서버 전체 상태를 실시간 모니터링

#### 세부 작업

| 주차 | 작업 | 산출물 |
|------|------|--------|
| 3주차 | Scouter Server 설치 및 Docker 컨테이너 구성 | `scouter/server.conf` |
| 3주차 | 각 Tomcat에 Scouter Java Agent 연결 | `scouter/agent.conf` |
| 3주차 | Scouter Client로 TPS, 응답시간, Active Service 모니터링 확인 | 스크린샷 |
| 4주차 | Prometheus + Node Exporter 구성 (CPU, Memory, Disk 수집) | `prometheus.yml` |
| 4주차 | Grafana 대시보드 구성 (WAS별 상태, 시스템 리소스 시각화) | 대시보드 JSON |
| 4주차 | JVM 힙 메모리, GC 현황, 쓰레드 덤프 분석 실습 | 분석 문서 |

#### 완료 기준
- Scouter에서 각 Tomcat 인스턴스의 실시간 성능 데이터 확인 가능
- Grafana 대시보드에서 서버 전체 상태를 한눈에 확인 가능

---

### Phase 3: 보안 연동 - SSL/PKI/SSO (5~6주차)

#### 목표
HTTPS 통신 + Keycloak SSO 통합 인증 환경 구현

#### 세부 작업

| 주차 | 작업 | 산출물 |
|------|------|--------|
| 5주차 | OpenSSL로 자체 CA 구축 및 서버 인증서 발급 | CA 인증서, 서버 인증서 |
| 5주차 | Nginx에 SSL/TLS 적용 (HTTPS 전환, 인증서 체인 설정) | SSL 설정 파일 |
| 5주차 | 인증서 갱신 자동화 스크립트 작성 | `cert-renew.sh` |
| 6주차 | Keycloak 설치 및 Realm, Client, User 설정 | `realm-export.json` |
| 6주차 | Spring Boot 앱에 Keycloak SSO 연동 (OIDC 프로토콜) | 앱 보안 설정 코드 |
| 6주차 | SSO 로그인 흐름 테스트 (한 번 로그인으로 다중 서비스 접근) | 인증 흐름 문서 + 스크린샷 |

#### 완료 기준
- HTTPS로 암호화된 통신 동작 확인
- Keycloak SSO로 통합 인증 로그인/로그아웃 정상 동작
- 자체 CA 인증서 체계를 설명할 수 있는 문서 완성

---

### Phase 4: 운영 자동화 및 장애 대응 (7~8주차)

#### 목표
실무 수준의 운영 자동화 및 장애 대응 역량 확보

#### 세부 작업

| 주차 | 작업 | 산출물 |
|------|------|--------|
| 7주차 | 일일 서버 점검 자동화 스크립트 (디스크, 메모리, 프로세스) | `health-check.sh` |
| 7주차 | 로그 분석 스크립트 (catalina.out, Nginx access/error log 파싱) | `log-analyzer.sh` |
| 7주차 | crontab 기반 정기 작업 스케줄링 (로그 로테이션, 백업) | crontab 설정 문서 |
| 8주차 | 장애 시나리오 #1: WAS 1대 다운 → Nginx 자동 페일오버 확인 | 장애 대응 보고서 |
| 8주차 | 장애 시나리오 #2: OOM 발생 → 힙 덤프 분석 및 JVM 튜닝 | JVM 튜닝 문서 |
| 8주차 | 장애 시나리오 #3: DB 커넥션 풀 고갈 → 원인 추적 및 조치 | 트러블슈팅 문서 |
| 8주차 | 전체 환경 Docker Compose 최종 정리 (IaC 완성) | 최종 `docker-compose.yml` |

#### 완료 기준
- 3가지 장애 시나리오별 원인 분석 → 조치 → 재발 방지 문서 완성
- 운영 자동화 스크립트 모음이 실행 가능한 상태
- `docker-compose up` 한 번으로 전체 환경 구동 가능

| 리스크 | 영향 | 대응 방안 |
|--------|------|-----------|
| Docker 환경 리소스 부족 (RAM 16GB 미만) | 다수 컨테이너 동시 구동 불가 | 컨테이너별 메모리 제한 설정, 필요 시 단계적 구동 |

| Keycloak SSO 연동 복잡도 | OIDC 설정 오류로 인증 실패 | 공식 문서 기반 단계별 설정, Spring Security 가이드 참조 |
| 자체 CA 인증서 브라우저 경고 | 신뢰할 수 없는 인증서 경고 표시 | 브라우저에 자체 CA 수동 등록, 문서에 절차 명시 |
